## 2. 结果解读

- 完整训练的 `baseline`（20 epoch）在 mAP50 与 mAP50-95 上均显著优于 10-epoch 的短训结果，这是预期的：更长训练能稳步提升精度。基线 mAP50≈0.787，mAP50-95≈0.460，是当前最强结果。
- `wiou_test` 与 `eca_test` 在 10 epoch 时均优于 `short_baseline` 的 10 epoch 表现（mAP50 分别 ~0.657 vs 0.631），表明 WIoU 与 ECA 在短期训练内能带来方向性改进。
- `wiou_test` 與 `eca_test` 的结果彼此接近，说明单独引入 WIoU 或 ECA 在短训下都有正向效果，但还需在相同（更长）训练预算下复现比较以校准结论。

## 3. 风险与注意事项

- 当前对比并非完全同配置：`baseline` 为 20 epochs，而 `wiou_test`/`eca_test`/`short_baseline` 为 10 epochs（短训）。要得出稳健结论，应在相同 epoch 数及超参下比较。
- 数据集包含空标签（已确认为正常情况），这会影响召回 / mAP 的绝对值，但对不同模型的相对比较影响较小，只要保持相同数据配置即可。

## 4. 推荐的下一步实验（按优先级）

1. 在相同训练预算（建议 20 epochs）下复现 `wiou_test` 与 `eca_test`，以便与 `baseline` 公平比较（关键指标：mAP50 / mAP50-95 / Precision / Recall）。
2. 并行或随后运行 FPN / PANet 变体短训（10–20 epochs），与 baseline/WIoU/ECA 在相同设置下比较。
3. 若第1步确认 WIoU 或 ECA 优势，组合试验：`baseline + WIoU + ECA`（同样先短训，再扩展到 50+ epoch）以检验协同效果。
4. 自动化对比：编写脚本聚合各 run 的 `results.csv`，生成对比图（mAP 与 loss 随 epoch 的曲线）并输出总结表格，便于定量展示差异。
5. 当候选模型稳定后，导出最佳权重并做推理基准（ONNX/TorchScript），测量推理速度与内存占用。
